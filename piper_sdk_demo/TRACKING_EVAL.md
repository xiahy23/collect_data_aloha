# Piper 双臂 Replay 跟踪精度评测方法

## 概述

本文档描述对 Piper 双臂机械臂执行 HDF5 数据回放（replay）时，评测末端跟踪精度的完整流程。评测分为两个阶段：**数据采集阶段**（replay 时同步记录实际关节角）和**离线分析阶段**（用正运动学计算末端误差）。

---

## 第一阶段：Replay 时记录实际关节角

### 工具

`collect_data/piper_sdk_demo/replay_on_slave.py`

### 原理

每回放一帧指令关节角之后，立即通过 `GetArmJointMsgs()` 采样两条机械臂的**实际**关节角，记录格式为：

```
shape: (T, 12),  dtype: float32,  单位: 0.001 度
列顺序: [左臂 j1, j2, j3, j4, j5, j6,  右臂 j1, j2, j3, j4, j5, j6]
```

采样在每帧插值发送完成后立即执行（即包含插值子帧的最后 CAN 报文发出后），因此记录的是机械臂在该时刻的真实响应位置。

### 输出文件

回放结束后自动保存至 HDF5 同级目录，名称格式为：

```
episode_N_actual.npy
```

### 相关参数

| 参数 | 默认 | 说明 |
|---|---|---|
| `--no-log-actual` | （未设置） | 加此标志则**不**保存 actual.npy |
| `--speed` | 50 | 关节运动速度，影响跟踪滞后 |
| `--interp` | 1 | 两帧之间的插值数，影响运动平滑性 |
| `--fps` | 30 | 回放帧率 |

## 第二阶段：离线 FK 误差分析

### 工具

`collect_data/piper_sdk_demo/eval_fk_error.py`

### 依赖

```
pip install numpy h5py matplotlib   # matplotlib 可选（--plot 时需要）
```

### 基本用法

```bash
# 最简调用（自动查找同名 _actual.npy）
python3 eval_fk_error.py --hdf5 ~/data/my_task/episode_0.hdf5

# 指定 actual 文件
python3 eval_fk_error.py \
  --hdf5   ~/data/my_task/episode_0.hdf5 \
  --actual ~/data/my_task/episode_0_actual.npy

# 只分析右臂 + 绘图
python3 eval_fk_error.py --hdf5 ... --arm right --plot

# 关闭滞后补偿搜索（仅输出原始误差）
python3 eval_fk_error.py --hdf5 ... --max-lag 0
```

### 所有参数

| 参数 | 默认 | 说明 |
|---|---|---|
| `--hdf5` | 必填 | 采集的 HDF5 文件路径 |
| `--actual` | 自动推断 | 实际关节角 .npy 文件路径 |
| `--arm` | `both` | 分析哪条臂：`left` / `right` / `both` |
| `--max-lag` | 60 | 滞后补偿最大搜索帧数；设 0 禁用 |
| `--fps` | 30.0 | 帧率，用于将 lag 换算为毫秒 |
| `--plot` | 关闭 | 保存误差曲线图为 `*_fk_error.png` |

## 正运动学（FK）实现

脚本内置纯 NumPy 实现的 Piper 6-DOF FK，参数直接来源于：

```
piper_ws/src/piper_description/urdf/piper_description.urdf
```

运动链：`base_link → j1 → j2 → j3 → j4 → j5 → j6 → tool_tip`

各关节参数（URDF xyz / rpy / 旋转轴方向）：

| 关节 | xyz (m) | rpy (rad) | 轴方向 |
|---|---|---|---|
| j1 | [0, 0, 0.123] | [0, 0, −π/2] | +z |
| j2 | [0, 0, 0] | [π/2, −0.10095, −π/2] | +z |
| j3 | [0.28503, 0, 0] | [0, 0, 1.3826] | +z |
| j4 | [0.021984, 0.25075, 0] | [−π/2, 0, 0] | −z |
| j5 | [0, 0, 0] | [π/2, −0.087266, 0] | +z |
| j6 | [0, 0.091, 0.0014165] | [−π/2, −π/2, 0] | −z |
| EE (固定) | [0, 0, 0.13503] | [π/2, 0, π/2] | — |

零位（所有关节 = 0）时末端位置约为 **(190 mm, 1 mm, 196 mm)**（相对 base_link）。

## 输出指标说明

### 原始误差（实时跟踪误差）

```
末端位置 RMSE       ← 所有帧末端三维欧氏距离的均方根，单位 mm
末端位置 最大误差   ← 最差帧的末端误差
末端位置 均值       ← 算术均值
<5mm 帧占比         ← 末端误差低于 5mm 的帧比例
<10mm 帧占比
<20mm 帧占比
关节角误差 (deg)    ← 每关节的均值/最大值，最大值超过 10° 时给出警告
```

**含义**：量化机械臂在**运动过程中每一帧**能多准确地到达指令位置。

### 滞后补偿误差（最终定位能力）

脚本在 `[0, max_lag]` 帧范围内遍历时移量 lag，找使 RMSE 最小的 lag，输出：

```
最优 lag=N 帧 / Xms    ← 最优时移量（机械臂固有响应延迟）
RMSE (lag补偿)         ← 补偿延迟后的末端误差 RMSE
最大误差 (lag补偿)
<5mm / <10mm 帧占比 (lag补偿)
```

**含义**：若补偿后误差明显小于原始误差（< 50%），说明主要误差来源是**跟踪滞后**（速度太快），机械臂本身定位能力尚可，可通过降低 `--speed` 或 `--fps` 改善；若补偿后误差改善有限，说明存在**系统性定位偏差**（零位漂移、标定误差等）。

## 典型结果解读（episode_0，右臂，speed=50）

```
末端位置 RMSE       : 24.76 mm
末端位置 最大误差   : 106.36 mm  (frame 460)
<10mm 的帧占比      : 38.2%

关节角误差 (deg):
  j2 最大: 18.5°,  j5 最大: 11.7°   ← 主要跟踪滞后来源

滞后补偿误差（最优 lag=3帧 / 100ms）：
  RMSE (lag补偿)  : 14.39 mm
  → 补偿后改善有限，可能存在定位偏差
```

解读：
- 3 帧（100ms）的系统延迟符合 CAN 总线通信 + 关节伺服响应的典型值
- 补偿延迟后 RMSE 仍有 14mm，说明存在一定系统性误差（非纯速度问题）
- j2、j5 误差大，建议在采集高速动作时适当降低速度或增大插值帧数

## 常见问题

| 现象 | 原因 | 处理方式 |
|---|---|---|
| `episode_N_actual.npy 不存在` | replay 时加了 `--no-log-actual` 或 replay 未完成 | 重新运行 replay（不加该标志） |
| 左臂 RMSE 异常小（接近 0） | 该 episode 左臂几乎未运动 | 正常，参考右臂数据 |
| `⚠ 关节未跟随指令` 警告 | 指令变化幅度大但实际几乎不动 | 检查 CAN 连接、enable 状态、speed 参数 |
| RMSE 超过 50mm | 速度过快或假零帧未完全过滤 | 降低 `--speed`，或检查 HDF5 前若干帧是否为零 |
